Conceptos clave
¿Qué es una API?
La analogía del bar
Tus amigos
tu aplicación
Cada uno tiene su pedido. No van individualmente a la barra.
El amigo mensajero
la API
Recoge los pedidos, va a la barra, habla en el idioma correcto, y vuelve con todo.
El barman
Claude
No atiende a todos a la vez. Solo entiende pedidos en el formato correcto.
La API no hace el café. No sirve la cerveza. Solo sabe cómo hablar con el barman — en el idioma correcto, en el formato correcto, con las instrucciones exactas que él entiende. Sin API, cada aplicación hablaría con Claude de una manera diferente. Con API, hay un único protocolo.
¿Qué es una clave API?
Tu carnet de identidad en el bar. El barman pregunta: "¿De parte de quién?" El amigo muestra el carnet. El barman sabe quién eres, si tienes cuenta, y cuánto puedes consumir.
sk-ant-api03-XXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXX
Es única, personal y no se comparte públicamente. Nunca la pongas en un repositorio público de GitHub ni la envíes por email. Quien la tenga puede usar tu cuenta y gastar tu dinero.

Prompt Engineering
¿Cómo construir
un buen prompt?
ROL
"Eres un experto
en mercado laboral..."
TAREA
"Analiza este CV
y estas ofertas..."
FORMATO
"Responde ÚNICAMENTE
con este JSON..."
Cuando falta una de las tres partes, el output se vuelve impredecible. Claude lee el prompt de arriba a abajo, prioriza lo que está primero, y aplica las restricciones como filtro al final.

El rol activa el contexto correcto.
La tarea define qué tiene que hacer.
El formato controla cómo tiene que responder.
Los 5 principios
1
Define el rol antes que la tarea
✕ "Analiza estas ofertas de trabajo"
✓ "Eres un experto en mercado laboral español... Analiza..."
2
Sé específico con los criterios
✕ "Calcula si encajo con la oferta"
✓ "Score 0-100: skills (40%), exp (25%), rol (20%), condiciones (15%)"
3
Controla el formato del output
✕ "Dame los resultados"
✓ "Responde ÚNICAMENTE con este JSON, sin texto adicional"
4
Usa ejemplos cuando puedas
✓ "nivel": "Junior|Mid|Senior|Lead" ← elige uno de estos cuatro exactos
5
Pon los límites al final
IMPORTANTE: No inventes datos · Si no hay salario, usa null · Máx 10 palabras

Prompt Engineering
El prompt del workshop,
explicado
Eres un experto en análisis del mercado laboral español,
especializado en perfiles de datos y tecnología.
ROL
Activa el contexto de mercado laboral español, no genérico.
Tu tarea es analizar un CV y un conjunto de ofertas
y devolver un análisis completo en formato JSON estricto.
TAREA + FORMAT HINT
Ya avisa desde el inicio que el output será JSON.
Lee el CV: extrae skills, años de experiencia,
seniority, idiomas, tipo de roles buscados.
INSTRUCCIÓN
Define exactamente qué extraer del CV.
Calcula un score (0-100) basado en:
  - Skills técnicas (40%)
  - Experiencia requerida (25%)
  - Tipo de rol (20%) · Condiciones (15%)
CRITERIOS
Con pesos explícitos. Sin esto, el score sería arbitrario.
RESPONDE ÚNICAMENTE con este JSON exacto,
sin texto adicional, sin markdown.
RESTRICCIÓN
"Únicamente" es clave — sin ella Claude añade texto extra.
¿Qué pasa si modificas el prompt?
Modificación Efecto esperado
skills técnicas → 60% Favorecerá más las coincidencias técnicas exactas
+ "valora especialmente el salario" Bajará el score de ofertas con salario bajo o sin indicar
+ "responde en inglés" Todo el output en inglés
+ criterio "cultura de empresa" Claude intentará inferirlo del texto de la oferta
"10 palabras" → "5 palabras" Recomendaciones más cortas y telegráficas
"El prompt no es la magia.
El prompt es la especificación."
